    Onboarding Documentation

    Structured criteria defining what winery information is public-facing versus private and kept for admin verification only.

    Data Segregation Policy

    All winery data is classified into two tiers. Public details are displayed on the Wineries page and VineMap to all visitors. Private details are collected for verification purposes and are never exposed in front-end code or accessible to non-admin users.

    Public Details

    • Winery name and estate identity
    • Region, country, and municipality
    • Primary grape variety
    • Classification (GG / 1G / Organic / Global)
    • Soil type and terroir description
    • VineMap coordinates (lat / lng)
    • Winery hero image and card emoji
    • Booking availability and experience types
    • Tasting experience prices
    • Sustainability and organic certifications (label only)

    Private Details (Admin Only)

    • Business registration number (HRB / SIREN)
    • VAT identification number
    • Legal entity documentation and articles of incorporation
    • Bank account / IBAN for payout processing
    • VDP / AOC / DOC certification scan
    • Full organic certification body + certificate number
    • Owner / manager legal name and personal contact
    • Direct phone and private email for communications
    • Proof-of-ownership documents (land title, lease)
    • Insurance policy reference number
    Winery Registration Process
    1. Winery owner registers with a corporate email domain (e.g. @winery.com).
    2. Admin receives notification and opens the Winery Management section to add the listing.
    3. Winery owner provides private proof documents via the Document Upload section.
    4. Admin verifies each item in the Verification Checklist and marks it complete.
    5. Once all 5 checklist items are ticked, the listing is promoted to public status.
    6. Public winery details become visible on winery.html and vinemap.html.
